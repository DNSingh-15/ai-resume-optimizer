# AI Resume Optimizer — Backend Architecture 🧠

This backend powers the **AI Resume Optimizer**, an intelligent system that analyzes resumes against job descriptions and provides:

* Accurate **ATS (Applicant Tracking System) score**
* **Missing keyword detection**
* **AI-powered improvement suggestions**
* **Optimized resume output**

The system uses a **multi-agent AI pipeline** combined with deterministic scoring logic to ensure that the ATS score remains **accurate, reliable, and explainable**.

---

# 🧠 System Architecture

The backend follows a **modular agent-based architecture** built with **FastAPI**.

Instead of relying entirely on LLM reasoning, the system separates responsibilities across **specialized agents** and **guardrails**.

This ensures:

* Deterministic ATS scoring
* Safe input/output validation
* Clean AI orchestration
* Production-ready architecture

---

# 🔁 Agentic Pipeline Flow

The resume analysis pipeline follows this flow:

```
User Resume + Job Description
            │
            ▼
      Input Guardrail Agent
  (Validate inputs & prevent misuse)
            │
            ▼
     Resume Parsing Agent
 (Structure resume text if needed)
            │
            ▼
    Keyword Extraction Agent
 (Extract keywords from resume + JD)
            │
            ▼
      ATS Scoring Engine
 (Deterministic ATS score calculation)
            │
            ▼
    Resume Optimization Agent
 (LLM generates improvements)
            │
            ▼
     Output Guardrail Agent
 (Validate response structure)
            │
            ▼
        Final Response
```

Key rule of the system:

**LLM never calculates the ATS score.**

The ATS score is always generated by a **deterministic scoring engine**, ensuring consistent results.

# 🤖 Agent Responsibilities

## 1. Input Guardrail Agent

This agent validates the incoming request before any processing occurs.

Responsibilities:

* Ensure resume text exists
* Validate job description input
* Prevent extremely large inputs
* Protect against prompt injection patterns

This prevents the LLM from receiving unsafe or malformed data.

---

## 2. Resume Parsing Agent

This agent structures raw resume text into a readable format for downstream agents.

Responsibilities:

* Normalize resume text
* Detect sections such as:

  * Experience
  * Skills
  * Education
* Prepare structured input for keyword analysis

---

## 3. Keyword Extraction Agent

This agent extracts keywords from both the resume and job description.

Responsibilities:

* Identify technology keywords
* Identify skill phrases
* Detect relevant role terms
* Prepare keyword sets for ATS scoring

Example extracted keywords:

```
["react", "node", "typescript", "aws", "docker"]
```

---

## 4. ATS Scoring Engine

This is the **most critical component** of the system.

The ATS score is computed **mathematically**, not by AI.

Responsibilities:

* Calculate keyword match percentage
* Identify missing keywords
* Produce deterministic ATS score

Example formula:

```
ATS Score = (Matched Keywords / JD Keywords) * 100
```

Example result:

```
Matched Keywords: 15
JD Keywords: 20

ATS Score = 75
```

This guarantees **consistent and accurate scoring**.

---

## 5. Resume Optimization Agent

This agent uses the LLM to generate improvement suggestions.

Responsibilities:

* Analyze missing skills
* Suggest resume improvements
* Provide optimized resume structure

The LLM only generates **recommendations and improvements**.

It never affects the ATS score.

---

## 6. Output Guardrail Agent

This agent validates the final AI response.

Responsibilities:

* Ensure valid JSON output
* Validate response schema
* Prevent hallucinated fields
* Enforce consistent response structure

If the response is invalid, the request fails safely.

---

# 🔒 Guardrails

The system includes **two layers of guardrails**.

## Input Guardrails

Protect against:

* Empty resumes
* Large payload attacks
* Prompt injection attempts
* Invalid job descriptions

## Output Guardrails

Protect against:

* Invalid JSON responses
* Missing required fields
* LLM hallucinations

# ⚙️ Running the Backend

Create a virtual environment:

```
python3 -m venv venv
```

Activate the environment:

```
source venv/bin/activate
```

Install dependencies:

```
pip install -r requirements.txt
```

Run the server:

```
python run.py
```

or

```
uvicorn app.main:app --reload
```
